关于Git和Github是什么

BTW，我最近真的忙疯了，但是我告诉自己必须坚持把这个玩意给做了。所以语言会有一些混乱，请见谅orz

作为一个已经玩了Git两年但还是要经常Google来解决一些问题的软件工程学生。如果要一句话概括Git的用途那就是：

他是一个适合程序员体质的同步笔记软件

怎么理解这句话呢，意思就是他的用途就好比命令行里的飞书云文档（当然他肯定比飞书云文档更早，而且他也可以有GUI界面），支持多端共享，多人协作，最最重要的是他强大的版本控制和分支管理能力。你想去到哪个节点，就去到哪个结点。就像时光机一样酷。

那么Github是什么呢？Github，Gitee还是Gitlab，其实都差不多，殊途同归，所以使用哪个作为远程仓库并无所谓，相比之下，使用Github或许会接触到更多有意思的人和世界各地的技术大牛？Github是一个可玩性很强的平台，而且他的logo真的很可爱（碎碎念）在这里你甚至可以学会做饭：

Git使用

如果是已经掌握了git的用法的同学，相信下面的命令对你来说就跟玩儿一样。如果是没有掌握的，光看这张表，相比也看不懂。

Git 命令 描述
git config 配置用户信息和偏好设置
git init 初始化一个新的 Git 仓库
git clone 克隆一个远程仓库到本地
git status 查看仓库当前的状态，显示有变更的文件
git add 将文件更改添加到暂存区
git commit 提交暂存区到仓库区
git branch 列出、创建或删除分支
git checkout 切换分支或恢复工作树文件
git merge 合并两个或更多的开发历史
git pull 从另一仓库获取并合并本地的版本
git push 更新远程引用和相关的对象
git remote 管理跟踪远程仓库的命令
git fetch 从远程仓库获取数据到本地仓库，但不自动合并
