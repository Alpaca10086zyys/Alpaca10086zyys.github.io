使用SSH连接开发机

什么是SSH？

SSH全称Secure Shell，中文翻译为安全外壳，它是一种网络安全协议，通过加密和认证机制实现安全的访问和文件传输等业务。SSH 协议通过对网络数据进行加密和验证，在不安全的网络环境中提供了安全的网络服务。

SSH 是（C/S架构）由服务器和客户端组成，为建立安全的 SSH 通道，双方需要先建立 TCP 连接，然后协商使用的版本号和各类算法，并生成相同的会话密钥用于后续的对称加密。在完成用户认证后，双方即可建立会话进行数据交互。

远程连接的好处就是，如果你使用的是远程办公，你可以通过SSH远程连接开发机，这样就可以在本地进行开发。而且如果你需要跑一些本地的代码，又没有环境，那么远程连接就非常有必要了。

然后在Web控制台开发机页点击SSH连接来查看登录命令：(我的开发机在我写这篇文章时已经停机，所以我用教程里的指令来进行详细解释)

这条 SSH 命令用于通过特定端口远程连接到服务器。以下是每个部分的解释：

  1. ssh：命令行工具，用于通过 SSH（安全外壳）协议连接到远程计算机。

  2. -p 37367：指定使用端口号 37367 进行连接（默认情况下，SSH 使用端口 22）。

  3. root@ssh.intern-ai.org.cn：远程主机信息，root 表示用户名，ssh.intern-ai.org.cn 是远程服务器的地址。

  4. -o StrictHostKeyChecking=no：此选项用于禁用主机密钥检查。如果此项设置为 no，SSH 不会提示用户验证主机的身份，即使密钥未知。

  5. -o UserKnownHostsFile=/dev/null：指定将用户已知的主机密钥文件设置为 /dev/null，即不记录远程主机密钥。这意味着不会保存连接的服务器信息，也不会在未来访问中被使用。

总结：这条命令连接到 ssh.intern-ai.org.cn 服务器的端口 37367，并通过 root 用户进行身份验证。它禁用主机密钥检查，并忽略记录已知的主机密钥，通常用于避免提示或为不受信任的环境创建临时连接。

需要注意的是：

主机密钥检查里的“主机”指的是服务器，而不是你的 PC 端。

当你通过 SSH 连接到一台服务器时，SSH 客户端会从该服务器接收到一个主机密钥，用来验证你正在连接的服务器的身份。
如果是你第一次连接这台服务器，SSH 会提示你是否信任并保存该主机密钥，以便在未来访问时验证服务器的身份。
这一步是为了防止中间人攻击，如果主机密钥发生变化，SSH 会发出警告。
在你的命令中，通过设置 -o StrictHostKeyChecking=no，SSH 跳过了这个验证步骤，因此不会询问你是否信任这台服务器的密钥。

端口映射

什么是端口映射？

端口映射是一种网络技术，它可以将外网中的任意端口映射到内网中的相应端口，实现内网与外网之间的通信。通过端口映射，可以在外网访问内网中的服务或应用，实现跨越网络的便捷通信。

在后续的课程中我们会进行模型web_demo的部署实践，那在这个过程中，很有可能遇到web ui加载不全的问题。这是因为开发机Web IDE中运行web_demo时，直接访问开发机内 http/https 服务可能会遇到代理问题，外网链接的ui资源没有被加载完全。

为了解决这个问题，我们需要对运行web_demo的连接进行端口映射，将外网链接映射到我们本地主机，我们使用本地连接访问，解决这个代理问题。

如何进行端口映射

在开发机选项卡，点击自定义服务，复制第一条命令（这里实际上是第二条也行，除非你机器上的7860端口已经被使用了，127.0.0.1对应的是本地机，localhost等同于127.0.0.1，区别在于localhost是域名，127.0.0.1是IP地址，它们都是本机访问，访问时都不需要联网）

命令各部分的含义：

  • p 37367：是指定 SSH 连接的端口为 37367，这个前面提到过。
    root@ssh.intern-ai.org.cn：表示要以 root 用户身份连接到 ssh.intern-ai.org.cn 这个主机。
  • CNg：
    • C 通常用于启用压缩。
    • N 表示不执行远程命令，仅建立连接用于端口转发等。
    • g 允许远程主机连接到本地转发的端口。
  • L {本地机器_PORT}:127.0.0.1:{开发机_PORT}：这是设置本地端口转发，将本地机器的指定端口（由 {本地机器_PORT} 表示）转发到远程主机（这里即 ssh.intern-ai.org.cn）的 127.0.0.1 （即本地回环地址）和指定的开发机端口（由 {开发机_PORT} 表示）。
  • o StrictHostKeyChecking=no：关闭严格的主机密钥检查，这样可以避免第一次连接时因为未知主机密钥而产生的提示或错误。

Linux入门指令

Linux在入门使用的时候最常用的就是下面几个：

  • pwd: 查看当前目录
  • cd: 切换路径（change directory）
    • cd ..:切换到上一级目录（相对路径）
    • cd /home:切换到home目录（绝对路径）
  • ls: 打印当前目录下所有的内容（文件夹和文件颜色是不一样的）
  • mkdir: 创建文件夹，在后面跟文件夹名，如果文件夹名中含空格，需要带引号
    • mkdir "dir name"
  • touch: 创建文件，在后面跟文件名，要带后缀，如果含空格与上一条同理
    • touch hello_world.py
  • cat: 查看文件内容，在后面跟文件名，将文件内容打印在终端。
  • vi: 使用vim编辑器创建（如果没有这个文件）并打开文件，进行编辑：
    • 编辑模式有多种，教程很多，我这里肯定说不清楚，事实上我掌握的也只是最基础的功能，建议是不会就去google，越用越熟吧
  • rm: 删除一个文件
    • rm hello.py
  • rm -r: 递归删除一个文件夹
    • rm -r "dir name"
  • grep: 跟一个字符串， 筛选出含有这个字符串的内容
    • grep "hello" file.txt
  • >: 将输出写入文件，覆盖原有内容
    • python hello.py > output.txt
  • >>: 将输出写入文件，追加到原有内容后面
    • python hello.py >> output.txt
  • |: 管道操作符，用于将一个命令的输出传递给另一个命令
    • python hello.py | grep "Hello"
